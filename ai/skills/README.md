# Skills

[README principal](../../README.md) | [Indice docs](../../docs/README.md) | [Volver a ai](../README.md)

Skills representan capacidades reutilizables para ejecutar o revisar trabajo con un criterio repetible.

## Regla de uso
- Usa una `skill` cuando quieres mantener la misma forma de producir o validar salidas entre multiples features.
- La skill no reemplaza juicio humano ni documentos oficiales.
- Debe reforzar trazabilidad, completitud y consistencia entre fases.
- Una skill oficial debe decir tambien cuando no aplicarla, como ejecutar el flujo y como verificar su salida.

## Anatomia minima recomendada
- objetivo,
- aplicala cuando,
- no la apliques cuando,
- entradas minimas,
- flujo recomendado,
- anti-rationalizations,
- red flags,
- verification evidence,
- referencias.

## Uso en proyecto real
- Las skills oficiales del proyecto deben reforzar la forma de trabajo real del equipo y sus artefactos canonicos.
- Si una skill nace desde la plantilla, adaptala para que apunte al dominio, rutas y criterios efectivos del proyecto adoptado.
- Una skill no debe quedarse validando ejemplos o rutas que ya no representan el repositorio real.
- Si la necesidad no esta clara, empieza por `using-project-skills.skill.md`.

## Anti-patron a evitar
- Mantener skills que todavia asumen el caso canonico o un stack distinto al del proyecto.
- Usar skills como sustituto de documentos oficiales o decisiones de arquitectura.

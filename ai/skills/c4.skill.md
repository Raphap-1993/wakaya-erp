# Skill C4

## Objetivo
Mantener consistencia al traducir arquitectura textual en vistas C4 con LikeC4.

## Aplicala cuando
- la arquitectura cambia,
- se agregan integraciones o contenedores,
- hace falta alinear documento tecnico y modelo visual.

## No la apliques cuando
- solo hace falta texto narrativo sin cambio estructural,
- el problema aun no tiene arquitectura base definida.

## Entradas minimas
- arquitectura base,
- actores externos,
- contenedores y relaciones clave.

## Criterios que debe reforzar
- nombres estables,
- relaciones claras,
- coherencia entre markdown y LikeC4.

## Flujo recomendado
1. Identifica que nivel C4 necesita el cambio.
2. Revisa relaciones, limites y actores relevantes.
3. Actualiza la vista minima necesaria.
4. Verifica alineacion con arquitectura y ADR.

## Anti-rationalizations
| Excusa | Respuesta |
|---|---|
| Con un diagrama generico basta | La vista debe responder una pregunta real |
| Nadie mira el C4 | Si existe como artefacto oficial, debe mantenerse coherente |

## Red flags
- El diagrama contradice arquitectura o ADR.
- Se agregan detalles irrelevantes al nivel C4 elegido.
- Cambia la arquitectura y el diagrama no se actualiza.

## Verification evidence
- vista C4 actualizada,
- coherencia con `03.00` y ADR revisada,
- nivel C4 elegido justificado.

## Salidas tipicas
- contexto,
- contenedores,
- notas de decision relevantes.

## Problema
Los equipos que gestionan reservations necesitan consultar reservationss de trabajo, cambiar estados con reglas claras y conservar historial auditable. Sin una solucion central, la trazabilidad depende de controles manuales, correos y revisiones tardias.

## Objetivo
Implementar Wakaya ERP como sistema web con API segura para operar reservations desde una reservations, registrar cambios de estado y consultar historial de auditoria.

## Alcance
- Gestion de reservations.
- Consulta y filtros por estado, prioridad y responsable.
- Cambio de estado con autorizacion RBAC.
- Historial de auditoria visible.
- Autenticacion OIDC y autorizacion por roles.
- Observabilidad basica para API y frontend.

## No alcance
- Motor BPM completo.
- Analitica avanzada de procesos.
- Migracion historica masiva.
- Firma digital avanzada.
- Integraciones externas no priorizadas para el MVP.

## Stakeholders
- Sponsor: area responsable de gestion documental.
- Product Owner: responsable funcional de Wakaya ERP.
- Business Analyst: responsable de reglas y trazabilidad.
- Tech Lead: responsable de implementacion.
- Security Lead: responsable de OIDC, RBAC y auditoria.
- Operaciones/SRE: responsable de despliegue, monitoreo y continuidad.

## Restricciones
- Stack base: node-next.
- API: /api/reservations.
- IAM: OIDC con issuer https://iam.wakayaecolodge.com/realms/wakaya-erp.
- Contenedor principal: ghcr.io/raphaelparedes/wakaya-erp.
- DR: tier-2 con RTO 4h y RPO 1h.

## Metricas de exito
- 95% de consultas de reservations responden en menos de 2 segundos.
- 100% de cambios de estado quedan auditados.
- 0 cambios de estado sin rol autorizado.
- Disponibilidad objetivo inicial de 99.5%.

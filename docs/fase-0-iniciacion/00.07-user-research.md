<a id="objetivo"></a>
## Objetivo
Integrar evidencias de usuario reales en el ciclo de descubrimiento y en cada iteracion, para reducir la brecha entre el problema percibido y el problema experimentado.

<a id="cuando-investigar"></a>
## Cuando investigar
- Antes de abrir una feature grande: validar problema y alternativas.
- Despues de liberar: medir fricciones reales y recoger feedback cualitativo.
- En rachas de hipotesis: cuando el equipo debate sin datos nuevos.

<a id="metodos-recomendados"></a>
## Metodos recomendados

| Metodo                    | Para que sirve                          | Esfuerzo |
|---------------------------|-----------------------------------------|----------|
| Entrevistas 1:1           | entender contexto y dolor               | medio    |
| Pruebas de usabilidad     | detectar fricciones en flujos           | medio    |
| Analisis de soporte       | recuperar friccion recurrente           | bajo     |
| Encuestas con NPS/CSAT    | medir percepcion a escala               | bajo     |
| Prototipos clickables     | validar hipotesis antes de construir    | medio    |
| Estudios diario           | entender uso real durante varios dias   | alto     |

<a id="plantilla-de-hallazgo"></a>
## Plantilla de hallazgo
```
Titulo: <hallazgo en una linea>
Fuente: <entrevista, analytics, soporte, sesion de prueba>
Evidencia: <cita, captura, metrica>
Severidad: alta / media / baja
Hipotesis: <que esta pasando y por que>
Accion sugerida: <experimento, cambio, seguimiento>
```

Los hallazgos aprobados se publican en `docs/fase-0-iniciacion/user-research/YYYY-MM-<slug>.md` en proyectos reales.

<a id="etica-y-consentimiento"></a>
## Etica y consentimiento
- Consentimiento informado por escrito antes de grabar.
- Datos sensibles anonimizados antes de salir del equipo de research.
- Retencion maxima de grabaciones segun politica (ver `docs/transversal/90.16-privacidad-compliance.md`).

<a id="checklist"></a>
## Checklist
- [ ] Cada iniciativa importante tiene al menos 3 evidencias cualitativas o cuantitativas citadas.
- [ ] Los hallazgos se revisan en review de producto y se priorizan con trazabilidad.
- [ ] La investigacion se programa con tiempo suficiente antes del kickoff.

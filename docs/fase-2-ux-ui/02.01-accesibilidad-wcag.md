<a id="objetivo"></a>
## Objetivo
Asegurar que las interfaces producidas desde esta plantilla cumplan como minimo WCAG 2.2 nivel AA, con practicas explicitas en diseno, implementacion y QA.

<a id="nivel-requerido"></a>
## Nivel requerido
- MVP: WCAG 2.2 AA en los flujos criticos (login, alta de expediente, bandeja, transicion de estado).
- Escala: AAA en componentes criticos (lectura continua, formularios largos, soporte con lector de pantalla).
- Extension: soporte de idiomas adicionales cuando el dominio lo requiera.

<a id="principios-wcag"></a>
## Principios WCAG
1. Perceptible: contraste >= 4.5:1 para texto normal; alternativas textuales para imagenes y multimedia.
2. Operable: toda accion accesible por teclado; foco visible; evitar trampas de foco.
3. Entendible: lenguaje claro; mensajes de error asociados a su campo; formatos consistentes.
4. Robusto: HTML semantico; ARIA solo cuando no hay elemento nativo equivalente.

<a id="checklist-por-pantalla"></a>
## Checklist por pantalla
- [ ] Landmarks (`header`, `nav`, `main`, `aside`, `footer`) presentes y unicos.
- [ ] Cada `button` y `a` tiene texto o `aria-label` descriptivo.
- [ ] Contraste validado con axe o Lighthouse; sin violaciones criticas.
- [ ] Navegacion completa con teclado (Tab/Shift+Tab/Enter/Space).
- [ ] Mensajes de error asociados via `aria-describedby`.
- [ ] Sin movimientos o parpadeos automaticos > 3 Hz.
- [ ] Soporte para `prefers-reduced-motion` cuando hay animaciones.

<a id="herramientas"></a>
## Herramientas
- Pruebas automaticas: `@axe-core/playwright`, `eslint-plugin-jsx-a11y`, `pa11y`.
- Pruebas manuales: lector de pantalla (NVDA, VoiceOver), navegacion con teclado, zoom 200%.
- Auditoria periodica con Lighthouse en CI.

<a id="objetivo"></a>
## Objetivo
Proveer una capa de design tokens y componentes base para los stacks web de la plantilla, de modo que cada proyecto parta con consistencia visual y pueda escalar a un design system completo.

<a id="tokens-canonicos"></a>
## Tokens canonicos
Los tokens viven en `stacks/*/template/src/shared/design-tokens/` (cuando se instancian) y se exportan en JSON, CSS custom properties y TypeScript.

| Categoria     | Ejemplo de token                | Uso                               |
|---------------|---------------------------------|-----------------------------------|
| Color primario| `color.brand.primary`           | botones primarios, enlaces        |
| Neutral       | `color.neutral.100..900`        | texto, fondos, bordes             |
| Tipografia    | `font.family.base`              | `system-ui, sans-serif`           |
| Tamano        | `font.size.sm/md/lg/xl`         | jerarquia tipografica             |
| Espaciado     | `space.xs/sm/md/lg/xl`          | margen, padding, gap              |
| Radios        | `radius.sm/md/lg`               | bordes redondeados                |
| Sombras       | `shadow.card/hover/modal`       | elevacion perceptible             |
| Motion        | `motion.duration/easing`        | transiciones y animaciones cortas |

<a id="componentes-base"></a>
## Componentes base
El set minimo por proyecto:

- Botones (primary, secondary, destructive, ghost) con estado `loading`.
- Inputs con label, helper text, error text y validaciones asociadas.
- Modal y Drawer con foco gestionado y cierre por `Escape`.
- Tabs, Breadcrumb, Pagination alineados con WCAG.
- Avatar, Badge, Tag, Tooltip.

<a id="distribucion"></a>
## Distribucion
- Opcion ligera: archivo `tokens.json` + generador `style-dictionary`.
- Opcion escalada: paquete `@org/design-system` publicado en registry privado.
- Cada release del design system incrementa SemVer y genera CHANGELOG propio.

<a id="checklist"></a>
## Checklist
- [ ] Tokens disponibles en CSS, JS y JSON.
- [ ] Cada componente base tiene historia en Storybook o equivalente.
- [ ] Pruebas visuales (percy, chromatic o playwright-visual) aprobadas.
- [ ] Accesibilidad de cada componente documentada y verificada.

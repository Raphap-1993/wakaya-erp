## Spec funcional
- [ ] Si la feature es visual, existe `gate-spdd-approved` aprobado o con observaciones aceptadas.
- [ ] El objetivo del feature esta claro.
- [ ] Existe actor principal y actores secundarios si aplica.
- [ ] Hay flujo principal.
- [ ] Hay flujos alternos o excepciones.
- [ ] Hay reglas de negocio y validaciones.
- [ ] Hay criterios de aceptacion verificables.
- [ ] Hay casos borde o restricciones relevantes.

## Spec tecnica
- [ ] La spec tecnica toma como entrada Product Design y SPDD cuando aplica.
- [ ] Existe trazabilidad al requerimiento o backlog item.
- [ ] Se define arquitectura objetivo o impacto en arquitectura.
- [ ] Se documentan contratos o interfaces.
- [ ] Se definen entradas, salidas y modelo de datos.
- [ ] Se documentan errores esperados.
- [ ] Se cubren seguridad y observabilidad.
- [ ] Se revisan responsabilidades, dependencias y excepciones contra [SOLID y diseno modular](../transversal/90.30-principios-solid-diseno-modular.md).
- [ ] Se documentan dependencias y riesgos.
- [ ] Se define estrategia de pruebas.
- [ ] Si UX define campos, acciones, validaciones, permisos o errores, existe `api-contract.md`.

## Tareas
- [ ] Las tareas derivan de la spec tecnica.
- [ ] Las tareas separan backend, frontend, QA y documentacion cuando aplica.
- [ ] Las tareas front/back derivan de SPDD aprobado cuando la feature es visual.
- [ ] Las tareas son trazables y ejecutables por un equipo.
- [ ] Cada tarea critica tiene objetivo, entradas y rutas permitidas.
- [ ] Cada tarea de codigo define ciclo TDD red-green-refactor.
- [ ] Cada tarea tiene comandos de verificacion y evidencia esperada.
- [ ] Cada tarea frontend enlaza UX, prototipo o mapping Penpot -> Angular cuando aplica.

## Validacion
- [ ] Las pruebas cubren criterios de aceptacion.
- [ ] Las dudas abiertas o decisiones pendientes quedaron registradas.
- [ ] La feature esta lista para entrar a construccion.
- [ ] El proveedor IA puede ejecutar la siguiente tarea sin inferir alcance ni rutas.
- [ ] La matriz `traceability.md` conecta Product Design -> SPDD -> SDD -> Construccion -> QA.

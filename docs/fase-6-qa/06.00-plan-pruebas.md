<a id="objetivo"></a>
## Objetivo
Definir la guia minima para planificar pruebas, evidencias y criterios de salida antes de liberar una feature o release.

<a id="nota-de-uso"></a>
## Nota de uso
Usa este documento como puente de fase para entender el minimo esperado. El plan oficial del proyecto debe vivir en `qa/fase-6-qa/plan-pruebas.md` y reflejar alcance, specs y criterios de aceptacion reales.

<a id="como-instanciar-esta-fase-en-un-proyecto-real"></a>
## Como instanciar esta fase en un proyecto real
- Toma `plantillas/fase-6-qa/plan-pruebas.md` como apoyo de trabajo, pero consolida la version oficial en `qa/fase-6-qa/plan-pruebas.md`.
- Si existen casos, matrices, evidencias o defectos en archivos separados, enlazalos desde `qa/fase-6-qa/`.
- No cierres la fase solo con este documento de guia; la evidencia real debe quedar fuera de `docs/`.

<a id="contenido-minimo"></a>
## Contenido minimo
- Objetivo y alcance de la validacion.
- Tipos de prueba a ejecutar.
- Criterios de entrada y salida.
- Casos o escenarios criticos.
- Ambiente de pruebas.
- Responsables y evidencias esperadas.

<a id="tipos-de-prueba-recomendados"></a>
## Tipos de prueba recomendados

| Tipo                  | Herramienta sugerida           | Donde vive en la plantilla | Obligatoriedad |
|-----------------------|---------------------------------|-----------------------------|----------------|
| Unitarios             | Vitest / JUnit                  | `stacks/*/template/src/**`  | Si             |
| Integracion           | Vitest + Testcontainers / JUnit | `stacks/*/template/`        | Si             |
| End-to-end            | Playwright                      | `stacks/*/template/e2e/`    | Si             |
| Accesibilidad (a11y)  | Axe + Playwright, Lighthouse CI | `stacks/node-next/template/tests/a11y/` | Si (AA) |
| Contratos             | Pact (consumer-driven)          | `stacks/node-next/template/tests/contract/` | Si por interfaz |
| Carga                 | k6                              | `stacks/node-next/template/tests/load/`   | Recomendado antes de release |
| Mutation testing      | Stryker / PIT                    | nocturno                    | Opcional       |
| Seguridad aplicativa  | ZAP baseline, semgrep            | `.github/workflows/security.yml` | Si          |

## Resultado

El backoffice editorial abre con cinco módulos de trabajo y el Home usa una
lista lateral más un editor enfocado. La vista previa, el historial, la
configuración web y la tipografía avanzada aparecen solo bajo demanda.

`Cupos de bungalows` conserva cantidades físicas por categoría y descuenta
únicamente reservas confirmadas. No existen acciones, formularios, rutas ni
servicios operativos para crear o cancelar bloqueos. Las tablas y filas
históricas permanecen únicamente para auditoría y rollback.

No se ejecutó ningún despliegue a producción.

## Escenarios funcionales cubiertos

- entrada editorial con Home, Páginas, Experiencias, Galería y Bungalows;
- Home sin preview, historial ni menú global visibles inicialmente;
- configuración web y preview abiertos bajo demanda;
- Páginas con una selección activa y el historial bajo demanda;
- tabla de cupos con total 17, cinco categorías y una sola acción `Editar total`;
- rangos semiabiertos: una nueva estadía puede iniciar el día de checkout de la anterior;
- edición bilingüe con un solo idioma visible en CTA, cifras y testimonios;
- listas y cabeceras operativas sin IDs de slides, tipos internos, slugs, códigos de bungalow ni IDs de assets;
- drawer de Doble con total físico 2, sin mutación durante el smoke;
- endpoints retirados de bloqueos no aceptan operaciones;
- `/admin/inventory` redirige a `/admin/bungalow-capacity` con HTTP 307;
- disponibilidad pública responde sin códigos individuales, `unitId`,
  `suggestedUnitId`, `totalUnits`, datos de bloqueo ni mantenimiento;
- Home público responde HTTP 200 sin datos internos de inventario.

## Incidencia local resuelta

Turbopack conservó en caché una versión anterior de la página de cupos y llamó
temporalmente a `listBlocks`. El bind mount ya contenía el archivo correcto; se
reinició únicamente `wakaya-erp-web` y la recompilación local respondió HTTP
200 con el contrato vigente.

pero no se declara aprobado automáticamente. Producción sigue bloqueada hasta
la demostración local y una aprobación explícita posterior.

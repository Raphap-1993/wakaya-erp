## Resultado

El editor del Home ya no reduce una publicación inválida al mensaje general
`invalid_home_content_payload`. Ejecuta preflight con el schema vigente y
muestra todas las incidencias con bloque, idioma y campo.

Cada incidencia ofrece `Ir al campo`. El primer error se abre automáticamente,
activa ES/EN, mantiene visible el nombre estructural de la sección, enfoca el
control y aplica `aria-invalid`. La fila lateral afectada muestra
`Revisar · N campos`. Una mutación inválida no llega al endpoint.

No se modificó la API, el documento persistido ni producción.

## Escenario funcional cubierto

1. abrir el editor autenticado;
2. seleccionar `Historia` y vaciar `Español > Título`;
3. volver al slide `Wakaya Ecolodge`;
4. pulsar `Publicar cambios`;
5. comprobar cero solicitudes `PUT /api/admin/home-content`;
6. comprobar resumen `Historia · Español · Título`;
7. comprobar selección automática de `Historia`;
8. comprobar foco y `aria-invalid="true"` en `Título`;
9. corregir el valor y comprobar que el resumen desaparece en vivo.

## Riesgo residual

La navegación visual usa los labels operativos del formulario como anclas de
foco. El flujo principal, CTA y tipografía están cubiertos por pruebas; si se
renombra un label, su mapping debe actualizarse en el mismo cambio.

## Rollback

Antes de integrar, eliminar la rama `codex/home-validation-guidance`. Después
de integrar, revertir el rango del incremento con:

```bash
git revert --no-commit 3aa754a^..HEAD
git commit -m "revert: remove guided home validation"
```

No se requiere rollback de base de datos ni de contenido.

## Resultado

Las imágenes del Home ya no se envían directamente al endpoint legado. Al
seleccionar una foto de slider, el editor exige recortes `Desktop` y `Mobile`;
las secciones Historia, Frase destacada y Cierre exigen un recorte fijo.

Después de confirmar el recorte, el pipeline administrado genera master y
variantes WebP, guarda la referencia portable `/media/assets/...` y permite
publicar el documento. El schema rechaza otras rutas relativas como
`/uploads/manual.jpg`, evitando referencias fuera del almacenamiento
administrado.

No se modificó producción.

## Causa raíz corregida

El editor del Home llamaba `/api/admin/home-content/media` en cuanto el usuario
elegía el archivo. Ese endpoint no abría crop y devolvía una ruta relativa que
el schema de publicación rechazaba por exigir una URL absoluta. Por eso la
interfaz mostraba primero una subida exitosa y luego un error al publicar.

## Evidencia TDD

- Red de schema: Zod rechazó inicialmente una referencia administrada
  `/media/assets/...`.
- Green de schema: acepta HTTP(S) y rutas `/media/...` segmentadas y seguras;
  continúa rechazando rutas relativas arbitrarias.
- Red del helper: Vitest falló porque `home-media-upload.ts` no existía.
- Red de navegador: Playwright confirmó que la selección no abría
  `Recortes obligatorios`.
- Green: el helper construye el multipart canónico y selecciona `heroDesktop`
  o `detail`; Playwright recorrió crop, POST 201 y publicación PUT 200.

## Verificación automatizada

- Suite completa: 147 archivos aprobados y 2 omitidos; 433 pruebas aprobadas
  y 3 omitidas.
- `npm run typecheck`: aprobado sin errores.
- ESLint focalizado en schema, helper, editor y E2E: aprobado sin errores.
- `npm run build`: aprobado; 69 páginas generadas y standalone actualizado.
- Playwright autenticado de media y regresión de validación: 2/2 aprobado.
- Health posterior al reinicio: HTTP 200.

Con la foto de prueba, el pipeline produjo `heroDesktop.webp` de 1920×1080 y
224 KB, y `heroMobile.webp` de 1080×1350 y 177 KB. Ambos se sirvieron como
`image/webp` desde `/media/assets/...`.

## Escenario funcional cubierto

1. abrir el editor autenticado;
2. seleccionar una imagen del primer slide;
3. comprobar que el endpoint legado no se invoca;
4. completar los recortes Desktop y Mobile;
5. comprobar `POST /api/admin/content/media` con estado 201;
6. comprobar el mensaje de imagen optimizada;
7. publicar y comprobar `PUT /api/admin/home-content` con estado 200;
8. comprobar ausencia de alerta de validación.

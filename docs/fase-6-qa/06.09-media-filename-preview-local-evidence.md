## Resultado

El nombre original de la imagen se normaliza y persiste en
`media_asset.original_filename` como `text` nullable. Las superficies
administrativas muestran el nombre como botón inmediatamente después del
`POST /api/admin/content/media` y lo conservan después de recargar. El botón
abre una vista previa accesible con nombre completo, imagen WebP, foco en
`Cerrar`, cierre por `Cerrar`, `Escape` y backdrop, y mensaje controlado cuando
la imagen no carga.

La implementación común cubre Home, Experiencias, Galería y Bungalows. Páginas
no tiene selector de media y por tanto no expone botones de vista previa. La
publicación conserva la referencia portable `/media/assets/...`; no hay URLs de
`localhost` persistidas.

## Migración y persistencia local

Se validó que `.env.local` es el symlink al entorno local principal y que la
conexión apunta al loopback y a la base esperada:

```bash
readlink -f .env.local
# /Users/rapha/Projects/wakaya-erp/.env.local
# DATABASE_URL: hostname=127.0.0.1, database=wakaya-erp

export DATABASE_URL="$(sed -n 's/^DATABASE_URL=//p' .env.local)"
npm run migrate
# exit 0

psql "$DATABASE_URL" -Atc "select column_name||'|'||is_nullable||'|'||data_type from information_schema.columns where table_name='media_asset' and column_name='original_filename';"
# original_filename|YES|text
```

La migración `013_media_original_filename.sql` es aditiva y nullable; no se
ocultan errores de tabla ausente en el servicio. El test del servicio también
verifica que el `INSERT`/`UPSERT` incluya el nombre normalizado.

## Residuos y límites conocidos

Los E2E restauran Home y Galería, pero no hacen garbage collection de los
activos subidos. Al cierre local quedaron activos huérfanos locales en
`.data/wakaya-media/` (`33` directorios de assets y `99` archivos observados;
`76` filas en `media_asset`, `18` con nombres de prueba
`preview-*`/`galeria-*`). Es deuda de limpieza local explícita, no bloquea este
gate y no se copia a producción. No se ejecutó ninguna escritura de migración,
subida o publicación fuera de la base y el filesystem locales.

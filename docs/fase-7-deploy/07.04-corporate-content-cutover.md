# 07.04 Cutover — contenido corporativo administrable

## Predeploy

1. Confirmar `/api/health`, políticas públicas y PM2.
2. Crear `pg_dump` custom y snapshot del release.
3. Verificar SHA-256 de ambos artefactos.
4. Registrar tamaño del log de error previo.

## Deploy

1. Sincronizar únicamente los archivos del incremento.
2. Ejecutar build standalone en el VPS.
3. Aplicar `011_corporate_content.sql`.
4. Reiniciar `wakaya-erp` con `--update-env`.
5. Publicar la revisión inicial desde la API administrativa autenticada si la tabla está vacía.

## Smoke

- `/api/health` = 200.
- `/es/about`, `/es/faq`, `/es/testimonials`, `/es/hotel-policies`, `/es/contact` = 200.
- rutas equivalentes `/en/*` = 200.
- política contiene reglas 24/48 horas y marco legal vigente.
- contacto contiene dirección, dos teléfonos y horario.
- `/admin/content?tab=company` y API corporate-content responden con sesión válida.
- revisión publicada en PostgreSQL y PM2 online.
- log de error no crece durante el recorrido.

## Rollback

El código puede restaurarse desde `release.tgz` y reiniciarse con PM2. La migración es aditiva y el release anterior ignora la tabla nueva. No restaurar el dump sobre producción si existen escrituras posteriores; restaurarlo en una base aislada y reconciliar hacia adelante.

## Objetivo
Ayudar a definir una combinacion de escenario consistente, no una etiqueta aislada.

## Checklist por dimension
### Madurez
- [ ] El proyecto necesita salir rapido como MVP o requiere gobierno enterprise desde el inicio.
- [ ] El alcance funcional inicial esta acotado o exige cobertura extensa.
- [ ] El nivel de formalidad documental esta claro.

### Arquitectura
- [ ] El dominio puede vivir en un monolito modular o necesita despliegue independiente por dominio.
- [ ] El equipo tiene capacidad operativa para microservicios si esa opcion se evalua.
- [ ] La complejidad de integracion justifica separar servicios.

### Despliegue
- [ ] Existen restricciones regulatorias o de infraestructura que obliguen on-premise.
- [ ] La elasticidad o servicios gestionados aportan valor real.
- [ ] Un modelo hibrido agrega valor o solo complejidad.

## Salida esperada
Declara el escenario final con tres dimensiones, por ejemplo:
- `Madurez: MVP`
- `Arquitectura: Monolito`
- `Despliegue: On-premise`

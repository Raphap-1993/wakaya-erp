## Objetivo
Ayudar a justificar la seleccion de stack con criterios tecnicos, operativos y de equipo.

## Checklist
- [ ] El stack encaja con el escenario definido.
- [ ] El equipo tiene experiencia suficiente para sostenerlo.
- [ ] El stack cubre los RNF mas criticos.
- [ ] El costo operativo es aceptable para el contexto.
- [ ] Las integraciones internas o corporativas encajan bien.
- [ ] La estrategia de despliegue y soporte es viable para el equipo.

## Salida esperada
Documenta la decision con:
- stack elegido,
- razones de seleccion,
- riesgos aceptados,
- impacto en despliegue, observabilidad y soporte.

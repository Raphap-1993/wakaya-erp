## Base transversal sugerida
- `src/` para codigo fuente con base neutral en `backend/`, `frontend/` y `shared/`.
- `tests/` para pruebas automatizadas en `unit/`, `integration/`, `e2e/` y `fixtures/`.
- `qa/` para artefactos de calidad organizados por fase.
- `ops/` para artefactos operativos organizados por fase.
- `docs/` para documentos de definicion y gobierno.
- `specs/` para features bajo SDD.

<a id="guias-por-stack"></a>
## Guias por stack
Cada stack tiene una ficha de referencia en `stacks/README.md` y en su carpeta correspondiente.

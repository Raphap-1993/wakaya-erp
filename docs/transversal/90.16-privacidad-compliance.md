<a id="objetivo"></a>
## Objetivo
Fijar las practicas minimas para que cualquier proyecto derivado cumpla regulaciones de proteccion de datos y politicas corporativas de compliance.

<a id="marco-aplicable"></a>
## Marco aplicable
- GDPR (UE) y sus equivalentes locales (LGPD, LFPDPPP, HIPAA cuando aplica salud).
- Politicas internas registradas en `docs/transversal/90.01-gobernanza.md`.
- Acuerdos contractuales con clientes o con subprocesadores.

<a id="ciclo-de-vida-del-dato"></a>
## Ciclo de vida del dato
- Captura: formulario o integracion con consentimiento explicito registrado.
- Almacenamiento: base de datos cifrada en reposo (`03.06-modelo-datos.md`) y respaldo cifrado.
- Uso: acceso por rol minimo necesario, con log de auditoria.
- Transferencia: TLS 1.2+ obligatorio; cualquier ruta cross-border documentada.
- Retencion: plazos definidos por categoria en `PRIVACY.md` y `ops/data/retencion.md`.
- Eliminacion: borrado logico + purga programada, validada por un job observado.

<a id="evaluaciones-requeridas"></a>
## Evaluaciones requeridas
- DPIA cuando se introducen categorias sensibles (salud, financiero, menores). Plantilla: [dpia-evaluacion-impacto.md](../../plantillas/transversal/dpia-evaluacion-impacto.md).
- ROPA (Record of Processing Activities) vigente para cada tratamiento. Plantilla: [ropa-registro-actividades.md](../../plantillas/transversal/ropa-registro-actividades.md).
- Revision de subprocesadores antes de contratar nuevos proveedores.
- Auditoria de permisos trimestral.

<a id="audit-trail-aplicativo"></a>
## Audit trail aplicativo
- Toda accion sensible emite una entrada conforme al esquema [`contracts/events/audit-log.schema.json`](../../contracts/events/audit-log.schema.json).
- El audit log se almacena en un sistema append-only (ej. bucket con Object Lock o base de datos con WORM).
- No se sobrescribe ni se borra salvo por politica de retencion justificada; los borrados quedan registrados.
- Las entradas son consumibles por SIEM y por el flujo de eDiscovery.

<a id="checklist"></a>
## Checklist
- [ ] La coleccion de datos tiene base legal documentada (consentimiento, contrato o interes legitimo).
- [ ] Existen procedimientos operativos para los derechos ARCO.
- [ ] Los logs no exponen PII ni secretos.
- [ ] Hay un responsable de privacidad nombrado y contactable.

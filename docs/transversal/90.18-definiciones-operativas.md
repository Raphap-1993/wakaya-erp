<a id="objetivo"></a>
## Objetivo
Documentar las practicas operativas minimas para que el proyecto pueda entrar y mantenerse en produccion sin depender del conocimiento de una sola persona.

<a id="on-call-y-guardias"></a>
## On-call y guardias
- Rotacion semanal entre personas capacitadas; relevos documentados en `ops/fase-8-operacion/oncall.md`.
- Respuesta objetivo a alertas criticas: 15 minutos en horario laboral, 30 minutos fuera.
- Escalamiento: primer responder -> lider de servicio -> gerencia tecnica, con ventanas de 30/60/90 minutos.
- Handoff al cierre de turno: estado de incidentes, cambios pendientes, riesgos conocidos.

<a id="postmortem"></a>
## Postmortem
- Obligatorio para incidentes de severidad alta (S1/S2) en menos de 5 dias habiles.
- Formato estandar: `ops/fase-8-operacion/postmortems/YYYY-MM-DD-nombre-corto.md`.
- Secciones minimas: Resumen, Linea de tiempo, Impacto, Causa raiz, Acciones correctivas con responsable y fecha, Aprendizajes.
- Cultura blameless: foco en el sistema y los procesos, no en las personas.

<a id="gestion-de-cambios"></a>
## Gestion de cambios
- Cambios en produccion requieren PR aprobado y pipeline verde.
- Ventana estandar de despliegue: lunes a jueves 09:00-17:00 horario local.
- Cambios de alto riesgo requieren plan de rollback explicito y aviso al canal operativo.
- Los despliegues fuera de ventana requieren aprobacion adicional del lider de servicio.

<a id="slo-sli-y-error-budget"></a>
## SLO, SLI y error budget
- Los SLO vigentes viven en `ops/fase-8-operacion/slo.md`.
- Cuando el error budget se agota, se congelan features y se priorizan acciones de estabilidad.
- Revision trimestral por producto y plataforma.

<a id="rotacion-de-secretos"></a>
## Rotacion de secretos
- Credenciales de base de datos y API keys: rotacion automatica cada 90 dias.
- Tokens de integracion externa: rotacion cuando lo exija el proveedor o ante sospecha.
- Certificados TLS: renovacion via ACM o cert-manager con alertas 30 dias antes.

<a id="checklist"></a>
## Checklist
- [ ] Hay una rotacion de on-call publicada y en vigor.
- [ ] Cada servicio critico tiene runbook y dueno identificado.
- [ ] Los ultimos 3 postmortems publicados cumplen el formato y tienen acciones con fecha.
- [ ] El error budget del trimestre esta comunicado a producto.

# Decisiones UX - Prototype HTML5 ejemplo

## Decisiones
- Se usa un solo `index.html` autocontenido para abrir sin build ni backend.
- Se simula estilo empresarial sobrio, no una UI final.
- Se usa JavaScript vanilla para validar flujo, roles, estados y feedback.
- Se documenta como referencia para `/prototype --mode html5`.

## Limitaciones
- No hay backend real.
- No hay autenticacion real.
- No hay descarga real.
- No reemplaza Penpot si el proyecto requiere diseno formal.
- No reemplaza Angular productivo.

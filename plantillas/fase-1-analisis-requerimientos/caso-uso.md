# Caso de Uso

> **Plantilla (no es el entregable).** Destino: `varios`. Consolida el entregable en la carpeta destino que indica el README de esta carpeta.

## Nombre
Usa un nombre corto orientado a negocio que identifique el flujo principal.

## Actor principal
Indica quien ejecuta el caso de uso y desde que contexto operativo.

## Precondiciones
Documenta el estado previo necesario para que el flujo pueda empezar.

## Flujo principal
Describe la secuencia normal paso a paso desde la accion inicial hasta el resultado esperado.

## Flujos alternos
Incluye variantes permitidas, excepciones o desvio del flujo principal sin entrar aun al detalle tecnico.

## Postcondiciones
Explica como queda el sistema o proceso una vez terminado el caso de uso.

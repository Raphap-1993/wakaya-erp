# Historia de Usuario

> **Plantilla (no es el entregable).** Destino: `varios`. Consolida el entregable en la carpeta destino que indica el README de esta carpeta.

## ID
Usa un identificador trazable, por ejemplo `HU-01`, alineado con backlog o requerimientos.

## Como
Define el rol o tipo de usuario que expresa la necesidad.

## Quiero
Describe la capacidad o accion que el usuario necesita realizar.

## Para
Explica el valor esperado, beneficio o resultado de negocio.

## Criterios de aceptacion
Incluye condiciones verificables que negocio y QA usaran para aprobar la historia.

## Reglas de negocio
Documenta restricciones, validaciones o condiciones que afectan el comportamiento esperado.

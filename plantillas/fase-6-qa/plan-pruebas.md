# Plan de Pruebas

> **Plantilla (no es el entregable).** Destino: `varios`. Consolida el entregable en la carpeta destino que indica el README de esta carpeta.

## Objetivo
Explica que alcance valida el plan y que decision de calidad ayuda a soportar.

## Alcance
Delimita features, modulos, ambientes o releases cubiertos por este plan.

## Tipos de prueba
Enumera los niveles de validacion que se ejecutaran, por ejemplo unitarias, integracion, E2E o QA manual.

## Criterios de entrada
Define condiciones minimas para empezar a probar, como build estable, ambiente listo o specs aprobadas.

## Criterios de salida
Aclara que evidencia o nivel de calidad se necesita para considerar cerrada la validacion.

## Casos o escenarios criticos
Lista los escenarios de mayor riesgo o valor que deben cubrirse obligatoriamente.

## Ambiente de pruebas
Describe el entorno, datos, accesos y dependencias necesarias para ejecutar pruebas.

## Responsables
Indica quien prepara, ejecuta, valida y aprueba la evidencia QA.

## Evidencias esperadas
Documenta que archivos, capturas, reportes o registros deben quedar en `qa/`.

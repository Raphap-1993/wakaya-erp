# v12.1.0 - Cierre operativo: Day-0, CMDB, SLOs, DR, AuthZ y CI de instanciacion

[README principal](../README.md) | [Indice docs](../docs/README.md) | [Volver a releases](README.md)

## Resumen
`v12.1.0` cierra las brechas ALTO detectadas en [`revisiones/2026-04-24-revision-adaptacion-proyecto-real.md`](../revisiones/2026-04-24-revision-adaptacion-proyecto-real.md) y las bloqueantes marcadas en [`revisiones/2026-04-24-revision-post-v12-1.md`](../revisiones/2026-04-24-revision-post-v12-1.md). El foco es convertir la plantilla "enterprise-ready como marco" en una plantilla "enterprise-ready como producto para un equipo real", con instanciaciones concretas, runbooks operables y validacion automatica en CI.

## Destacado
- **Day-0 runbook humano**: `docs/fase-0-iniciacion/00.09-plan-arranque-15-dias.md` con cronograma dia por dia y aprobaciones.
- **CMDB raiz Backstage**: `catalog/` con `all.yaml`, `domains/`, `systems/`, `groups/`, `users/`, `templates/new-service.yaml`.
- **SLOs como codigo**: `ops/observability/slos.yaml` en formato OpenSLO v1 + generador Node `generate-slo-rules.mjs` (multi-window multi-burn-rate 14.4x/6x) + `docs/fase-3-arquitectura/03.07-slos-como-codigo.md`.
- **Rotacion de secretos operable**: `ops/runbooks/rotacion-secretos.md` + `ops/runbooks/rotate-secrets.mjs` con providers `vault`/`aws`/`gcp` y modos dry-run/apply/verify/retire-previous/restore.
- **Plan y runbook de DR**: `docs/fase-8-operacion/08.02-plan-dr.md` (Tier 0-3, RTO/RPO, 5 escenarios, GameDays) + `ops/runbooks/dr-failover-region.md` + `ops/runbooks/verify-dr.mjs`.
- **AuthN/AuthZ canonico**: `docs/fase-3-arquitectura/03.08-auth-authz.md` + instanciaciones en los 4 stacks:
  - `node-next`: `src/lib/rbac.ts` + `src/middleware/authn.ts` con `jose` + JWKS.
  - `java-monolith` y `spring-react`: `Permission`, `Role`, `Rbac`, `RbacEvaluator`, `SecurityConfig` con `spring-boot-starter-oauth2-resource-server`, method security y mapping de claim `roles`.
  - `quarkus-angular`: `quarkus-smallrye-jwt` + `quarkus-security` + equivalentes de Permission/Role/Rbac/RbacEvaluator y `SecurityInfo` para uso con `@RolesAllowed`.
- **Gestion de incidentes end-to-end**: `docs/transversal/90.24-gestion-incidentes.md` con severidades S1-S4, roles (IC, Comms, Ops, Scribe), flujo de vida, plantilla de postmortem blameless y metricas (MTTD/MTTA/MTTR).
- **Scripts cross-platform**: trio `.mjs` + `.sh` + `.ps1` para `bootstrap`, `init-project` y `new-service`. Zero dependencia de Python o Bash-only.
- **Validacion de config por schema**: `scripts/validate-template-config.mjs` + `scripts/schema/template.config.schema.json` con campos nuevos `auth.*`, `observability.*`, `dr.*`, `secrets.*`. `new-service.mjs` la invoca antes de copiar.
- **CI de instanciacion**: `.github/workflows/template.yml` valida schema, chequea tokens canonicos en plantilla y dry-run del `new-service.mjs` por los 4 stacks en `ubuntu-latest` + `macos-latest` + `windows-latest`.

## Compatibilidad
- Cambios aditivos en su mayoria: la estructura previa sigue funcionando.
- `template.config.example.json` exige ahora `auth`, `observability`, `dr`, `secrets`. Cualquier config existente debe extenderse o el schema la rechazara. El error es explicito con ruta del campo faltante.
- `scripts/new-service.mjs` ahora valida la config antes de copiar; no hay cambio de interfaz.
- Stacks JVM anaden dependencias de Spring Security (monolith, spring-react) y Quarkus Security + SmallRye JWT (quarkus-angular). El primer `mvn` o `gradle` tras el bump descarga estas transitive deps.
- `HealthControllerTest` de los stacks Spring usa ahora `@AutoConfigureMockMvc(addFilters = false)` para mantener la prueba verde con security activo. `HealthResource` de Quarkus anade `@PermitAll`.

# v12.10.0 - Design system y mapping Penpot Angular en Fase 2

[README principal](../README.md) | [Indice docs](../docs/README.md) | [Volver a releases](README.md)

## Resumen
`v12.10.0` endurece Fase 2 UX para que los prototipos Penpot no sean pantallas sueltas: ahora deben usar componentes reutilizables, estados consistentes, tokens base y mapping hacia Angular/Nx. Ademas, `create-project` deja Fase 2 lista desde la primera generacion del expediente y `/ux` queda como comando de refresco a demanda.

## Destacado
- **Sistema de componentes UX**: nuevo artefacto generado `docs/fase-2-ux-ui/02.07-sistema-componentes-ux.md`.
- **Mapping Penpot Angular**: nuevo artefacto generado `docs/fase-2-ux-ui/02.08-mapping-penpot-angular.md`.
- **Generacion inicial completa**: `create-project` genera fases 0-8 mas UX/Penpot/Angular mock desde el primer expediente.
- **Actualizacion no destructiva**: `create-project --refresh-existing` rehidrata documentacion y UX sobre un expediente ya creado sin recrear scaffolding.
- **Refresco UX idempotente**: `/ux` permite actualizar Fase 2 cuando cambian RF, HU, specs, pantallas o decisiones de componentes.
- **Penpot por RPC local**: nuevo `ops/prototyping/penpot/create-prototype-file.ps1` para crear usuario demo opcional, proyecto, archivo de prototipo y evidencia fechada.
- **Prompts especializados**: nuevo `ai/prompts/generated/componentes-penpot-angular.md`.
- **Skill de consistencia**: nuevo `ai/skills/design-system-mapping.skill.md`.
- **Gate UX Ready reforzado**: exige componentes reutilizables, variantes, estados y mapping antes de considerar listo el prototipo.

## Flujo actualizado
```text
Documentacion inicial
  -> create-project
  -> fases 0-8 + UX/Penpot/Angular mock
  -> RF/HU/specs evolucionan
  -> /ux
  -> UX Markdown
  -> Sistema de componentes UX
  -> Penpot clickable
  -> Mapping Penpot Angular
  -> Angular mock
```

## Regla clave
Ninguna pantalla Penpot debe usar elementos sueltos si existe un componente reutilizable aplicable. Si aparece un componente nuevo, debe agregarse al sistema de componentes y al mapping Penpot -> Angular.

# v12.11.0 - Delivery IA por proveedores

[README principal](../README.md) | [Indice docs](../docs/README.md) | [Volver a releases](README.md)

## Resumen
`v12.11.0` agrega una capa operativa para que proveedores IA no se queden sin saber que hacer al pasar de documentacion a ejecucion. El flujo ahora exige brainstorming cuando hay ambiguedad, planes pequenos, worktrees, TDD, revision de codigo, QA y cierre de branch.

## Destacado
- Nuevo flujo transversal: `docs/transversal/90.33-flujo-delivery-ia-proveedores.md`.
- Nueva referencia reusable: `ai/references/feature-delivery-workflow.md`.
- Nuevas skills: `brainstorming`, `using-git-worktrees`, `writing-plans`, `executing-plans`, `test-driven-development`, `requesting-code-review` y `finishing-development-branch`.
- `spec-tareas.md` ahora baja specs a tareas ejecutables por proveedor IA.
- `gate-4-6` exige evidencia de tareas, TDD, review y QA.
- `gate-7-8` exige rama/worktree cerrable, PR/merge o release con riesgos residuales.

## Flujo actualizado
```text
idea / RF / HU / spec
  -> brainstorming
  -> /spec
  -> writing-plans
  -> worktree/rama
  -> /build + TDD
  -> /review
  -> /test
  -> /ship
```

Resultado esperado:
- Task packets de fase 4-7 con flujo de delivery IA.
- Spec de tareas con IDs, rutas permitidas, TDD y comandos.
- Gate 4-6 y 7-8 reforzados.
- Validacion documental e instanciacion sin hallazgos.

## Compatibilidad
- Cambios aditivos.
- No obliga a usar worktrees si el equipo no los adopta, pero exige declarar rama/workspace y evidencia cuando haya codigo.
- No reemplaza aprobacion humana de negocio, arquitectura, QA o release.

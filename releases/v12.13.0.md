# v12.13.0 - Product Design, SPDD, SDD y construccion

`v12.13.0` corrige el orden metodologico para que el framework tenga algo mostrable y validado antes de construir.

## Orden oficial
```text
Product Design
  -> SPDD - Spec + Prototype Driven Development
  -> SDD - Spec-Driven Development
  -> Construccion Front + Back
```

## Cambios clave
- Nuevo gate `gate-spdd-approved`.
- Nuevas salidas por feature para Product Design, SPDD, prototipo, validacion, API contract, UI test cases y trazabilidad.
- SDD queda bloqueado para features visuales si no existe prototipo validado.
- Construccion front/back consume SPDD aprobado y specs tecnicas, no prototipos sueltos.

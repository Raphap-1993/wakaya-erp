# v12.14.0 - HTML5-first Prototyping

`v12.14.0` incorpora prototipado UX navegable como paso explicito entre SPDD y SDD.

## Cambios principales
- Se agrega `docs/fase-2-ux-ui/02.14-html5-first-prototyping.md` como guia oficial de prototipos HTML5 autocontenidos.
- Se agrega `/prototype` con modos `html5` y `penpot`.
- Se agrega `gate-prototype-ready` para separar prototipo revisable de prototipo validado.
- Se agregan skills de prototipado HTML5 y Penpot AI.
- Se refuerza que `gate-spdd-approved` requiere validacion humana.
- Se agrega criterio minimo UX para flujo, estados, validaciones, roles/permisos, datos mock, navegacion y feedback.
- Se agrega ejemplo HTML5 navegable en el caso canonico y ejemplo reusable en `ejemplos/fase-2-ux-ui/prototype-html5/`.
- Se endurece `AGENTS.md` como contrato operativo raiz para alinear lectura obligatoria, commands, agents, skills, prompts, references, quality gates y runbooks.
- Se alinea `90.32-agente-interno-framework-ai-first.md` con modos de trabajo, `/prototype`, gates de prototipo y la separacion intake/bootstrap/orquestacion.
- Se alinean `90.00`, `90.01`, `90.14`, `90.28` y `90.33` con el delivery IA actualizado, `/prototype`, `gate-prototype-ready` y `gate-spdd-approved`.
- Se limpia el recorrido end-to-end de IA/UX: `provider-orientation-pack.template.md`, referencias UX y plan de arranque quedan alineados con HTML5/Penpot; Angular Mock queda solo como legacy/excepcion.
- Se separan los modos de trabajo: intake desde fuente bruta, bootstrap desde template y orquestacion con agente IA.

## Reglas
- HTML5 valida rapido, no reemplaza Angular productivo.
- Penpot formaliza diseno cuando hay colaboracion, design system o handoff visual.
- No cerrar SDD ni iniciar construccion productiva de feature visual sin prototipo validado.
- `.tmp/` y bases `.db/.sqlite` no forman parte de entregables distribuibles.

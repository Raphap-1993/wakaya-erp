# v12.15.0 - Calidad de prototipos HTML5 medible

`v12.15.0` cierra la brecha por la que el agente de IA generaba prototipos HTML5 pobres mientras los ejemplos golden si alcanzaban nivel producto-real. El framework ahora no solo **describe** nivel 3: tambien lo **mide** y lo bloquea cuando no se cumple.

## Problema que resuelve

El sistema documentaba nivel 3 como aspiracion, pero su piso operativo estaba calibrado a nivel 1-2:
- El ejemplo "neutro" `ejemplos/fase-2-ux-ui/prototype-html5/` era una shell generica de 150 lineas que el agente copiaba como referencia oficial.
- El generador del template (`scripts/ai-framework-agent.mjs`) inyectaba ese mismo patron pobre en cada proyecto nuevo.
- El validador solo verificaba presencia de palabras clave; un prototipo pobre pasaba en verde.
- La plantilla era un checklist booleano que invitaba a hacer "lo minimo para marcar casillas".

## Cambios principales

### Referencias
- `ejemplos/fase-2-ux-ui/prototype-html5/` se renombra a `prototype-html5-anti-ejemplo/` con README que documenta cada bloqueo que acumula.
- Se agrega el tercer golden `prototype-html5-golden/dashboard-analytics-kpi/` (KPI con sparklines, grafico comparativo, donut, tabla drill-down, panel lateral, diferencias por rol) como referencia nivel 3 para analitica.

### Validador
- `ci/scripts/check-html5-prototype-quality.mjs` se reescribe con grading nivel 0-3.
- Automatiza los bloqueos B1, B2, B3, B4, B5, B6, B7 y B9.
- Exige minimos cuantitativos (lineas, tokens CSS, media queries, vistas, mock records, botones) calibrados contra los goldens reales.
- Valida los seis campos del pre-flight en `decisiones-ux.md`.

### Prompts, plantilla y comando
- `ai/prompts/generar-prototipo-html5-ejecutable.md` exige leer integro el golden del dominio, citarlo en `decisiones-ux.md` y cerrar con un bloque de auto-rating verificable.
- `plantillas/fase-2-ux-ui/prototype-html5.md` deja de ser checklist booleano y pasa a ser el pre-flight de decisiones con tabla de minimos por nivel.
- `ai/prompts/generar-prototipo-html5-desde-spdd.md` queda como redireccion al prompt ejecutable canonico.
- `ai/commands/prototype-command.md` agrega el paso obligatorio de correr el validador `--strict` y reportar el auto-rating; `gate-html5-product-quality.md` documenta que B1-B7 y B9 son verificacion automatica.

### Generador del template (raiz del problema)
- `scripts/ai-framework-agent.mjs` deja de inyectar el patron pobre.
- `html5PrototypeFiles(ctx)` emite un seed nivel 2 autocontenido (saas-operativo: topbar, sidebar, KPIs, filtros, tabla, estados, toast) y un `decisiones-ux.md` con el pre-flight en blanco.
- Se eliminan las dos emisiones duplicadas del patron pobre.
- Guarda CLI para que el modulo sea importable sin disparar `main()`.

### Showcase end-to-end
- `specs/001-bandeja-trabajo-expedientes/prototype-html5/` regenerada a nivel 3 (1498 lineas, 30 tokens, 7 vistas, 21 mocks, 25 botones).
- Se crea el hub `prototype/index.html` con journeys por actor, actores del sistema, prototipos por vertical, matriz de cobertura y estado de gates.

## Reglas

- El minimo para `gate-prototype-ready` sigue siendo nivel 2; nivel 3 es lo recomendado para enterprise.
- El generador entrega un seed nivel 2; subir a nivel 3 requiere ejecutar `/prototype --mode html5` con el golden del dominio.
- El validador es una red de seguridad calibrada: pasa los goldens reales, rechaza el anti-ejemplo, exige nivel 2 minimo. B8 y B10 siguen requiriendo revision humana.
- Ningun prototipo puede declararse "listo" sin el bloque de auto-rating ni sin pasar el validador en modo `--strict`.

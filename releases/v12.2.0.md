# v12.2.0 - ADRs, catalogo enriquecido, rotacion programada y verify-dr parametrizado

[README principal](../README.md) | [Indice docs](../docs/README.md) | [Volver a releases](README.md)

## Resumen
`v12.2.0` cierra las brechas MEDIO identificadas en [`revisiones/2026-04-24-revision-post-v12-1.md`](../revisiones/2026-04-24-revision-post-v12-1.md): registra como ADR las decisiones que entraron en v12.1, completa el catalogo Backstage con `API` y `Resource` kinds, automatiza la rotacion de secretos con un workflow scheduled, hace `verify-dr.mjs` apto para multiples ambientes con flags explicitos, y marca el cierre de las brechas ALTO heredadas de la revision post-v12.0. Es una release de consolidacion.

## Destacado
- **ADR-007 OpenSLO**: registra la decision de usar OpenSLO v1 como formato canonico de SLOs y documenta alternativas (Sloth DSL, Pyrra CRDs, Nobl9, reglas a mano) y consecuencias.
- **ADR-008 JWT + OIDC + RBAC**: deja explicita la decision de autenticacion (RS256/ES256 con JWKS) y autorizacion (RBAC con permisos como acciones sobre recursos), incluyendo trade-offs frente a session cookies, mTLS y tokens opacos.
- **ADR-009 DR multi-region activo-pasivo**: formaliza la clasificacion en Tier 0-3, los GameDays trimestrales y la consecuencia operativa de cada decision.
- **Catalog Backstage completo**: nuevos `catalog/apis/expedientes-rest.yaml` y `catalog/apis/expedientes-events.yaml` apuntando a `contracts/api/openapi.yaml` y `contracts/events/asyncapi.yaml`. Recursos `expedientes-db.yaml`, `expedientes-cache.yaml` y `expedientes-events-bus.yaml` referenciados desde `all.yaml`. La `Location` raiz ahora habilita `API`, `Resource`, `Component`.
- **Rotacion programada**: nuevo `.github/workflows/rotate-secrets.yml` con cron trimestral (primer lunes 03:00 UTC) en dev/staging y `workflow_dispatch` para prod. Autentica al provider via OIDC (AWS / GCP / Vault) sin secretos de larga vida; sube audit log a artefacto del run y, si esta configurado `vars.AUDIT_BUCKET`, lo replica a S3 con Object Lock.
- **verify-dr.mjs parametrizado**: nuevos flags `--kubeconfig`, `--namespace`, `--core-selector`, `--prometheus-url`, `--prometheus-token`, `--threshold-error-rate`, `--query`, `--dry-run`, `--json`. Separacion interna en providers `k8s` y `prometheus`. Permite usarlo desde CI sin editar codigo.
- **Runbooks README como tabla canonica**: `ops/runbooks/README.md` rehecho con tabla `runbook | trigger | oncall | script | frecuencia | RTO/RPO`, plantilla para nuevos runbooks y referencias a workflows asociados.
- **Cierre de brechas ALTO documentado**: el informe original `revisiones/2026-04-24-revision-adaptacion-proyecto-real.md` tiene ahora una seccion 9 con el mapping `brecha -> release que la cierra -> entregable concreto`.

## Compatibilidad
- Cambios aditivos. La cadena nav-guided ahora pasa por ADR-007 -> 008 -> 009 antes de la fase 4; `fase-4-sdd/README.md` actualiza su `Anterior`.
- El workflow `rotate-secrets.yml` permanece inactivo si no se configuran las variables `vars.SECRETS_PROVIDER`, `vars.AWS_ROTATE_ROLE_ARN` (o equivalente GCP/Vault) y `vars.AUDIT_BUCKET`. Sin esas variables el job falla limpiamente al primer paso de auth, sin tocar estado.
- `verify-dr.mjs` es retrocompatible: los flags previos `--region`, `--env`, `--post-failover`, `--kubectl-context`, `--prom-url` siguen funcionando. `--prom-url` queda como alias de `--prometheus-url`.
- El catalogo Backstage requiere extender `allow:` con `[API, Resource, Component]` en `app-config.yaml` para que las nuevas entradas se procesen. Documentado en `catalog/README.md`.

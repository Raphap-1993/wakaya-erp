`v12.23.0` agrega un panel de **ejecucion manual** de comandos del agente
dentro del front embebido (`memory-serve`): validadores, sync de memoria,
reportes y generadores se disparan desde la UI con consola live, sin tener
que abrir terminal. Todo bajo whitelist cerrado y bind a `127.0.0.1`.

## Que hay nuevo

### Tab "Acciones"

Nuevo tab al final de la barra de navegacion del front (al lado de "Busqueda").
Layout de 2 columnas:

- **Izquierda**: 4 categorias plegables con los botones de cada accion.
- **Derecha**: consola estilo terminal (tema oscuro) con stdout/stderr
  coloreado, exit code, duracion, botones Limpiar/Copiar.

### 17 acciones en 4 categorias

| Categoria      | Acciones                                                                                          |
|----------------|---------------------------------------------------------------------------------------------------|
| Memoria        | `sync-memory`, `index-docs`, `embed-docs`, `regenerate-context`, `harvest-trace`                  |
| Validadores    | `check-docs`, `check-trace-drift`, `check-trace-coverage`, `check-prototype-hub`, `check-prototype-html5` |
| Reportes       | `memory-report`, `context-pack`, `next-task`, `diff-since` (con arg), `template-drift`, `status` |
| Generadores    | `generate-prototype-hub` (danger), `generate-prototype-hub --auto-only`                           |

### Endpoints nuevos

```
GET  /api/actions   -> catalogo seguro (sin paths internos)
POST /api/exec      -> ejecuta una accion del whitelist
                       body: { id, arg? }
                       resp: { actionId, argv, exitCode, signal, durationMs,
                               stdout, stderr, timedOut }
```

Ejecucion sincrona; timeout default 5 min. La respuesta incluye stdout y
stderr completos (max 8MB cada uno).

### Refresh automatico del snapshot

Cuando la accion ejecutada modifica la BD (`sync-memory`, `index-docs`,
`embed-docs`, `regenerate-context`, `harvest-trace`), el cliente vuelve a
hacer `GET /api/snapshot` y re-renderiza todas las tabs (Trazabilidad,
Gates, Decisiones, etc.). El usuario ve el efecto del comando sin recargar.

## Seguridad (Fase 1 MVP, todas las reglas activas)

1. **Bind explicito `127.0.0.1`**: antes `memory-serve` escuchaba en todas
   las interfaces. Ahora solo loopback. Si necesitas acceso remoto: SSH
   tunnel o reverse proxy con auth.
2. **Whitelist cerrado**: `EXEC_ACTIONS` lista las 17 acciones con su `argv`
   fijo. El cliente solo envia el `id`; no se acepta `command`, `argv` ni
   shell strings.
3. **CSRF via Origin**: `POST /api/exec` exige cabecera `Origin:
   http://localhost:<port>` o `http://127.0.0.1:<port>`. Otros origins â†’ 403.
4. **Content-Type JSON**: sin `application/json` â†’ 415.
5. **Body limitado a 4KB**: prevencion basica de DoS.
6. **Validacion de `arg`**: las acciones con parametro (solo `diff-since`
   actualmente) declaran regex; arg que no matchea â†’ 400.
7. **One-job lock**: una accion a la vez; concurrentes â†’ 409 con el id en curso.
8. **Timeout 5 min**: `spawnSync` con `timeout: 5*60*1000`.
9. **Confirm modal para `danger:true`**: `generate-prototype-hub` (variante
   full) requiere confirmacion explicita antes de sobrescribir
   `prototype/index.html`. La variante `--auto-only` no la requiere porque
   preserva customizaciones fuera de `@auto:start`.

## Como usarlo

```bash
# Arranca el server (bindea solo a 127.0.0.1)
npm run memory:serve
# o
node scripts/ai-framework-agent.mjs memory-serve --port 4319

# Abre http://localhost:4319 en el navegador
# Click en el tab "Acciones"
# Click en cualquier accion -> se ejecuta y muestra la salida
```

Tambien funciona desde `curl`:

```bash
curl -X POST http://127.0.0.1:4319/api/exec \
  -H "Content-Type: application/json" \
  -H "Origin: http://localhost:4319" \
  -d '{"id":"sync-memory"}'
```

## Que NO esta en esta version (fase 2 y 3 futuras)

- **Streaming SSE**: la ejecucion es sincrona; el usuario ve la salida
  cuando termina (vs aparecer linea por linea). Funciona bien porque la
  mayoria de acciones tardan < 5s. Sera necesario cuando se integre
  `index-docs` sobre repos grandes (> 10k archivos).
- **Historial persistente** en `ai_action_runs`: por ahora la consola
  conserva solo lo de la sesion actual del navegador.
- **Atajos de teclado** (`Ctrl+Shift+S` = sync, etc.).
- **Boton "Detener"** para cancelar un job en curso (requiere streaming).

## Migracion desde v12.22

Sin migracion necesaria. La BD no cambia. El nuevo tab aparece en cualquier
proyecto que ya tenga `memory-serve` corriendo, una vez que se actualiza el
template y `scripts/ai-framework-agent.mjs`.

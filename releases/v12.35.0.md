# v12.35.0 — Phases granulares + check API/Test/BD-columnas + GH Actions + bug fix

`v12.35.0` cierra 7 pendientes acumulados (3 de coherencia + 4 de
validadores extendidos) en un release cohesivo. Tambien resuelve un
bug de sintaxis subtle introducido en check-test-documented por el
patron `**/` en un block comment.

## Lo nuevo

### 1. `check-trace-coverage` con phases granulares (4 vs 5 vs 6)

Antes era binario: fase >=5 exige codigo + test. Ahora:

| Fase | Codigo | Test | Severidad |
|---|---|---|---|
| 4 (SDD) | no exigido | no exigido | OK |
| 5 (Construccion) | obligatorio | recomendado (warning) | error si falta codigo |
| 6+ (QA, Deploy) | obligatorio | obligatorio | error si falta cualquiera |

El validador distingue `error` vs `warning` y exit-codes acorde. Permite
enmendar el reporte por fase real, no solo "esta o no esta en fase 5+".

### 2. `check-bd-documented` con columnas minimas

Antes solo exigia que apareciera `` Tabla `<nombre>` `` en algun
spec-tecnica.md. Ahora valida que el bloque tenga:

- Tabla markdown `| Columna | Tipo | Notas |` con al menos una fila.
- Declaracion de PK (palabra `PK` o `PRIMARY KEY` en algun lugar).
- Seccion `Indices: ...` con al menos un indice.

```
$ npm run check:bd-documented
BD INCOMPLETAS: 1 tabla(s) documentadas pero sin columnas minimas.
  - "rbac" en specs/001/spec-tecnica.md:
      * falta declaracion de PK (poner 'PK' en la fila del id o 'PRIMARY KEY' en restricciones)
      * falta seccion 'Indices: ...' con al menos un indice
```

### 3. `check-api-documented` nuevo (paridad con bd)

Para cada `target_type='api'` en `ai_trace_links`, verifica que exista
una marca `METHOD /path` en algun `api-contract.md` del repo (de la
feature o global).

Filtra valores que no son endpoints reales (ej. `401/403` que es
descripcion de respuestas, no endpoint).

Resultado en el template:
```
OK. 6 endpoints declarados, 7 entradas documentadas en api-contract.md. Cero huerfanos.
```

### 4. `check-test-documented` nuevo

Para cada `target_type='test'` en features con `phase >= 6`, verifica
que exista archivo real en `tests/`, `qa/`, `src/<feature>/`, etc.
Filtra placeholders (`-`, `tbd`) y descripciones (`e2e seguridad`,
`performance/smoke`).

En features con phase < 6 no se aplica (los nombres aspiracionales son
normales en SDD/construccion temprana).

### 5. `check-ai-artifacts` extendido con `## Modelo de datos` obligatoria

Cada `specs/<feature>/spec-tecnica.md` ahora debe tener la seccion
`## Modelo de datos`. Sin ella, falla CI.

### 6. Plantilla `api-contract.md` reescrita con regla v12.35+

Mismo patron que `spec-tecnica.md` de v12.34:
- Regla v12.35+ explicita.
- Ejemplo canonico `METHOD /api/<resource>`.
- Tabla de errores estandarizada.
- Caso "no expone API propia" para features de solo consumo.

### 7. GitHub Actions workflows

```yaml
# .github/workflows/template-checks.yml
on:
  pull_request:
    paths:
      - "ai/**"
      - "ci/**"
      - "docs/**"
      - "plantillas/**"
      - "specs/**"
      - "scripts/**"
      - "ejemplos/**"
      - "*.md"
      - "package.json"
jobs:
  template-checks:
    steps:
      - checkout, setup-node 22
      - npm ci --ignore-scripts
      - bootstrap memoria (init + index + sync)
      - npm run check:template

# .github/workflows/project-checks.yml
on:
  pull_request:
    paths:
      - "src/**"
      - "backend/**"
      - "frontend/**"
      - "tests/**"
      - "specs/**/traceability.md"
      - "specs/**/spec-tecnica.md"
      - "specs/**/api-contract.md"
      - "TRACEABILITY_MATRIX.md"
jobs:
  project-checks:
    steps:
      - npm run check:project
```

Ambos hacen bootstrap de memoria + paths-filter para no correr cuando
no aplica.

## Bug fix bonus: `**/` en block comment

`check-test-documented.mjs` tenia este comment:

```javascript
 * verifica que exista archivo real en `tests/`, `qa/`, `src/**/__tests__/`
```

El `**/` cerraba el `/* */` block comment prematuramente, causando un
SyntaxError "Unexpected identifier 'No'" en la primera linea de codigo
despues del comment. Fix:

```diff
- * src/**/__tests__/
+ * src/<feature>/__tests__/
```

**Leccion**: nunca usar `*/` dentro de un block comment, ni siquiera en
patrones como `**/__tests__/**`. Si necesitas hablar del patron glob,
escribelo con placeholder (`<feature>/__tests__/`).

## Migracion desde v12.34

Sin pasos manuales. Si tu proyecto real tiene matrices con tablas BD
sin columnas minimas, `npm run check:bd-documented` te dira exactamente
que agregar. Si tiene endpoints declarados sin doc en `api-contract.md`,
`npm run check:api-documented` te lo reporta.

Los GH Actions workflows se activan automaticamente cuando los archivos
listados en `paths:` cambian.

## Que NO esta todavia

- **`check-bd-documented` con verificacion de tipos compatibles**
  (cada columna usar tipos SQL conocidos: TEXT, UUID, INT, etc).
- **`check-api-documented` con OpenAPI/JSON Schema validation**
  (hoy solo verifica que el endpoint este declarado, no su shape).
- **`check-test-documented` con coverage minimo** (hoy verifica que
  existe el archivo, no que cubra el RF asociado).
- **Plantilla `spec-funcional.md` con seccion obligatoria** equivalente.
- **Workflows con concurrency cancel-in-progress activo por SHA**
  (hoy solo por ref).

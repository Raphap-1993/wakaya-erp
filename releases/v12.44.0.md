# v12.44.0 — Rebranding del paquete (@ndavilal/project-template-ai-first)

`v12.44.0` cierra la unica observacion menor que dejo la revision
externa de v12.43:

> "En package.json el nombre aparece como `@anthropic/project-template`.
> Si esto es solo interno no hay problema. Si lo van a distribuir como
> framework propio, convendria cambiarlo a algo neutral, por ejemplo:
> `@empresa/project-template-ai-first`."

El reviewer aclaro: `@empresa = ndavilal`.

## El cambio

```diff
- "name": "@anthropic/project-template",
+ "name": "@ndavilal/project-template-ai-first",
```

Una linea en `package.json`. Sin impacto funcional.

## Auditoria de consistencia

Antes del cambio, busque cualquier otra referencia para evitar
inconsistencias futuras:

```bash
$ grep -rln "@anthropic" project-template/
project-template/package.json    ← unica fuente, ya cambiada
```

No habia referencias en docs, workflows, scripts ni changelogs
historicos. Solo el `package.json`.

## Convencion documentada

Nueva seccion en `INSTANCIACION_PROYECTO_REAL.md`:

```markdown
## Convencion de nombre del paquete (v12.44+)

El `package.json` raiz del template usa el nombre
`@ndavilal/project-template-ai-first`. Cuando generes un proyecto real
con `create-project`, el `package.json` emitido usa
`@org/<feature-slug>` por default — reemplazalo por tu scope npm
real (`@empresa/<servicio>`) antes del primer push:

  { "name": "@miorg/expediente-api", "version": "0.1.0", ... }

Si publicas en un registry privado (Verdaccio, GitHub Packages, Nexus),
el scope debe coincidir con el namespace del registry.
```

Asi un agente IA que abre el repo entiende inmediatamente:
- Por que el template canonico usa `@ndavilal/`.
- Que los proyectos generados deben re-scopearse al `@empresa` real.
- La relacion con el registry privado.

## Sin impacto funcional

- Ningun script npm (`memory:bootstrap`, `check:all`, `prototype:hub`, etc.).
- Los 7 validadores de `check:project`.
- Los 4 validadores de `check:template`.
- El panel UI con sus 25 acciones, 13 endpoints, 17 presets.
- La memoria viva (267 docs, 2565 chunks/embeddings, 33 trace links).
- El generador `create-project` (ese siempre emite `@org/<slug>`, no `@anthropic/`).

## Migracion desde v12.43

Sin pasos manuales. Solo cambia el `name`. Si alguien dependia del
nombre anterior en su `package.json` (lo cual seria raro porque el
template no se publicaba a npm), debe actualizarlo.

## Que NO esta todavia

- **Publicar el template en npm/GitHub Packages** bajo el nombre nuevo
  para que sea instalable como `npm i -g @ndavilal/project-template-ai-first`.
- **Auto-rename del scope en `create-project`** desde el CLI
  (`--scope @miorg`).
- **README badge** con el nombre del paquete + version.
- **`engines` constraints en package.json** (hoy solo `node >=22.0.0`;
  podriamos agregar `npm >=10`).

# v12.49.0 — cierre de gaps post-v12.48 (BD path + scaffold prototipo + validador strict)

Release que cierra 7 gaps detectados al re-revisar v12.48 después de descubrir que opencode, codex y gemini terminaron con bases de datos en distintas rutas. La auditoria también identificó que **la causa raíz real de los prototipos pobres no era documentación, sino falta de scaffolding mecanizado** que fuerce el copy del golden.

## TL;DR

- **Bug grave del template**: `AGENTS.md` decia `.agent/` pero el código usa `ai/memory/`. Los 3 agentes terminaron en rutas distintas. **Resuelto**.
- **Causa raíz REAL de prototipos pobres**: agentes "leen el golden" pero generan desde cero con Tailwind CDN. **Cerrado** con `npm run scaffold:prototype` (auto-copia del golden) + 4 reglas nuevas en validador (B-CDN, B-MINIFIED, O-INLINE, O-NAV).
- 5 fixes documentales adicionales (pre-flight simplificado, scaffold en orden de lectura, @trace en componentes visuales, scaffold:prototype en npm scripts).

## El problema central

Al re-revisar v12.48 descubrimos dos clases de errores:

### Clase 1 — Contradicciones documentales

| Concepto | Fuente A (decía) | Fuente B (decía) | Resultado real |
|---|---|---|---|
| Ruta BD | `AGENTS.md:24` → `.agent/` | `scripts/ai-framework-agent.mjs:357` → `ai/memory/` | opencode creó BD en LAS DOS; codex en `.agent/`; gemini en `ai/memory/` |

Estas contradicciones convierten al template en "elige tu aventura" para el agente. v12.49 unifica la ruta canónica a `ai/memory/framework-agent.db` (default del código) y documenta `.agent/` como opcional con `--db` explícito.

### Clase 2 — Causa raíz real de prototipos pobres (no era documentación)

El primer audit dijo "agregar más documentación visual + obligar a leer el golden". v12.48 lo aplicó, pero un segundo audit profundo descubrió que **el problema cognitivo es diferente**:

> "El agente lee el golden, toma notas mentales, pero luego escribe HTML minificado con Tailwind CDN (43 líneas, 0 tokens CSS) en lugar de copiar-adaptar el golden (640 líneas, 14 tokens). La causa no es ignorancia del golden; es que el prompt no incluye un paso explícito de **scaffolding mecanizado** que fuerza al agente a copiar y hacer sustituciones textuales precisas."

v12.49 cierra este gap con:
- **`npm run scaffold:prototype --feature X --domain streaming`** que copia mecánicamente el golden de streaming + reemplaza title/marca/link al hub.
- **4 reglas nuevas en el validador** (B-CDN, B-MINIFIED, O-INLINE, O-NAV) que rechazan los antipatrones reales vistos en gemini.

### F3 — Validador `check-html5-prototype-quality.mjs` con 4 reglas nuevas

Cierra los antipatrones reales vistos en gemini (que pasaban silenciosamente):

#### `[B-CDN]` bloqueante — CDN externo prohibido

```javascript
const CDN_PATTERNS = [
  { re: /https?:\/\/cdn\.tailwindcss\.com/i, label: "Tailwind CDN" },
  { re: /https?:\/\/cdn\.jsdelivr\.net\/npm\/bootstrap/i, label: "Bootstrap CDN" },
  // ... 7 patrones más
];
```

#### `[B-MINIFIED]` bloqueante — HTML minificado

Detecta si la línea más larga > 1000 chars con > 20 tags. Cierra el antipatron real visto en gemini (002-catalogo de 43 líneas con HTML inline minificado).

#### `[O-INLINE]` observación — exceso de style="..." inline

Si hay > 20 `style="..."` inline pero < 30 declaraciones en `<style>`, alerta como observación (no bloqueante).

#### `[O-NAV]` observación — navegación decorativa

Detecta cuando hay `onclick=alert()/console.log` sin handlers reales (no usa `addEventListener`, `setView`, `render`, etc.).

### F4 — Pre-flight checklist simplificado (7 → 5 items)

Items 4, 6 y 7 del checklist v12.46 eran redundantes: `check-template-instantiation` strict ya cubre la verificación de los 9 archivos canónicos y slugs únicos. Bajamos de 7 a 5 items netos sin perder cobertura:

```bash
# 1. Sincronizar memoria
npm run memory:sync

# 2. Validadores del template (4/4 OK)
npm run check:template

# 3. Validadores del proyecto (10/10 OK)
npm run check:project

# 4. Strings residuales + slugs únicos + 9 archivos + scripts v12.45 + AI_CONTEXT zones
node ci/scripts/check-template-instantiation.mjs --mode instantiated --root .

# 5. Memoria sincronizada (exit 1 si stale)
node scripts/ai-framework-agent.mjs status --fail-on-drift --json | tail -5
```

### F5 — Scaffold commands documentados en AGENTS.md

Nueva sección "Comandos canonicos para generar artefactos (v12.49+)" con tabla de 6 comandos + flujo canónico de 5 pasos. Los 3 agentes reales nunca descubrieron `scaffold-feature` porque no estaba en AGENTS.md; ahora es lo primero que ven.

### F6 — `@trace` en componentes React/Vue/Angular/Svelte (regla explícita)

Nueva subsección en `docs/transversal/90.35-trace-annotations-por-stack.md`:

> **Regla obligatoria**:
> 1. Top del archivo (siempre): JSDoc del `default export` o componente principal.
> 2. Hooks/handlers internos: NO anotar (la anotacion del componente principal cubre todo el archivo).
> 3. Sub-componentes en otros archivos: cada archivo lleva su propio `@trace` en su default export.
> 4. Tests del componente: archivo de test también lleva `@trace` en su top.

Con ejemplos para React, Vue 3, Angular, Svelte.

### F7 — `scaffold:prototype` agregado a `package.json` (template + generador)

```json
"scaffold:feature": "node scripts/scaffold-feature.mjs",
"scaffold:prototype": "node scripts/scaffold-prototype.mjs",
```

Disponible desde el día 1 del proyecto sin pasos manuales.

# v12.6.0 - Cierre de drift "expediente" -> `reservation` y bug de instanciacion .java

[README principal](../README.md) | [Indice docs](../docs/README.md) | [Volver a releases](README.md)

## Resumen
`v12.6.0` cierra las brechas ALTO de [`revisiones/2026-04-26-revision-post-v12-5.md`](../revisiones/2026-04-26-revision-post-v12-5.md). El cambio mas importante es operativo: se corrigio un bug en `init-project.mjs` que omitia `.java` y `.html` del listado de extensiones procesadas, lo que hacia que las constantes Java de RBAC (`Permission.java`) quedaran con literal `reservation:read` tras la instanciacion. Tambien se sincronizan tests, doctrina y catalogo Backstage con la generalizacion del dominio iniciada en v12.4.

## Destacado
- **Tests Java reparados** (los 3 stacks JVM): `RbacTest.java` invocaba `Permission.EXPEDIENTE_READ`/`_WRITE`/`_APPROVE`, constantes que el enum ya no expone (paso a `RESOURCE_*`). El codigo Java no compilaba; ahora si.
- **`init-project.mjs` extiende `TEXT_EXTENSIONS`**: anade `.java`, `.jsx`, `.html`, `.css`, `.scss`, `.dockerfile`, `.env`. Sin esto, los archivos Java emergian post-instanciacion con `reservation:read` literal, lo que provocaba drift cross-stack invisible al `check-rbac-consistency.mjs` (que solo compara el repo template).
- **`check-template-instantiation.mjs` extiende su `TEXT_EXTENSIONS`** con las mismas extensiones para que el job `dry-run-new-service` detecte tokens residuales en archivos Java/HTML/CSS post-instanciacion.
- **Catalog Backstage tokenizado**: `catalog/users/ejemplo.yaml`, `catalog/groups/team-expedientes.yaml`, `catalog/systems/expedientes.yaml`, `catalog/apis/expedientes-{rest,events}.yaml`, `catalog/resources/expedientes-{db,cache,events-bus}.yaml` ahora usan `team-wakaya`, `wakaya-erp`, `Wakaya ERP` y `wakaya-erp-db`. Los nombres de archivo se mantienen estables (Backstage lee `metadata.name`, no el filename), y `catalog/all.yaml` sigue referenciando los filenames originales.
- **Doctrina sincronizada con codigo**: `docs/fase-3-arquitectura/03.08-auth-authz.md` ahora muestra la matriz `reservation:read|write|approve` y enumera ejemplos de instancias reales (`case:read`, `customer:read`).
- **ADR-008** actualizado para usar `reservation:approve` en el ejemplo `@PreAuthorize` y mencionar `check-rbac-consistency.mjs` como guardian del contrato cross-stack.
- **Comentarios obsoletos eliminados** en `RbacEvaluator.java` (3 stacks), `SecurityConfig.java` (2 stacks Spring), `authn.ts` y `logger.ts` (node-next).

## Compatibilidad
- Cambios aditivos en su mayoria.
- `init-project.mjs` ahora procesa **mas archivos**. Si tu config personalizada coincide accidentalmente con un placeholder en archivos `.java`/`.html`/`.css`, esos se sustituiran. Revisa el repo generado tras el primer `new-service.mjs`.
- El catalogo Backstage requiere ejecutar `init-project.mjs --root .` sobre el clone del project-template antes de cargarlo. Si tu equipo carga `catalog/` directamente desde el repo project-template sin instanciar, vera tokens literales (`wakaya-erp`).

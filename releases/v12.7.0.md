# v12.7.0 - Agente interno enterprise AI-first y documentacion real generada

[README principal](../README.md) | [Indice docs](../docs/README.md) | [Volver a releases](README.md)

## Resumen
`v12.7.0` convierte la capa IA del template en una entrada operativa: el agente interno puede leer la documentacion del framework, enrutar intenciones por fase, recomendar comandos, crear proyectos reales y dejar artefactos documentales canonicos validados.

## Destacado
- **Agente interno ejecutable**: `scripts/ai-framework-agent.mjs` incluye memoria SQLite local, indexacion de docs, busqueda, routing, `/document`, `plan-create` y `create-project`.
- **Comando `/document` first-class**: `ai/commands/document-command.md` formaliza la entrada documental para ideas, necesidades, requerimientos, decisiones y cambios de alcance.
- **Gate documental transversal**: `ai/quality-gates/gate-documentation-ready.md` valida fase, ruta canonica, objetivo, entradas, supuestos, preguntas abiertas, trazabilidad y siguiente paso.
- **Skill de orquestacion documental**: `ai/skills/documentation-orchestration.skill.md` conecta fases, commands, gates, references, prompts, agents y rutas canonicas.
- **Routing corregido**: requerimientos claros se mandan a Fase 4 SDD con `/document + /spec`; decisiones de stack se mandan a Fase 3 con ADR; ideas iniciales se quedan en Fase 0-1.
- **Proyecto real AI-provider-ready**: `create-project` genera documentacion real del dominio tras crear el stack tecnico, incluyendo indices visibles por fases 0-8, operacion IA por fase, task packets, agents, commands, skills, gates, prompts y manifest para proveedor IA.

## Prueba real ejecutada
Comando validado:

```powershell
node scripts/ai-framework-agent.mjs create-project --stack quarkus-angular --config config/expediente.config.json --dest C:\template\caso-real1\expediente --skip-smoke --no-git --force
```

Resultado:
- 78 artefactos operativos generados para `Gestion de Expedientes`.
- 77 archivos Markdown y `ai/provider-manifest.json`.
- README real con backend `expediente-api`, frontend `expediente-web`, API `/api/expedientes` y paquete `com.acme.expediente`.
- Requerimientos con actores, RF/RNF, reglas de negocio, integraciones y backlog inicial.
- Arquitectura, decisiones tecnologicas, ADR-001, despliegue, specs, QA, runbook, rollback y operacion.
- Indices visibles en `docs/fase-0-iniciacion/` hasta `docs/fase-8-operacion/`, apuntando a los entregables canonicos.
- Operacion IA por fase en cada README de fase.
- Task packets listos en `ai/tasks/` para ejecutar las fases con proveedor IA.
- `check-docs` sobre el proyecto generado: 77 Markdown sin hallazgos.
- `check-template-instantiation --mode instantiated`: sin hallazgos.
- Escaneo propio del agente contra frases residuales de plantilla: sin hallazgos.

## Compatibilidad
- Cambios aditivos.
- El comando `create-project` ahora escribe documentacion adicional en el proyecto destino y falla si quedan senales de plantilla.
- La busqueda semantica depende de `sqlite-vec` externo; la memoria SQLite y busqueda textual funcionan sin dependencias adicionales.

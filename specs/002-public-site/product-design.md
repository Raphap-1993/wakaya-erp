# Product Design - Wakaya ERP Public Site

[README principal](../../README.md) | [Specs](../README.md)

## Problema
La web publica actual de Wakaya tiene valor visual, pero la nueva iteracion
corre el riesgo de caer en un sitio generico si no se fija una direccion clara
antes de construir.

## Objetivo
Prototipar una web publica premium para Wakaya Ecolodge que:

- venda hospedaje primero
- se vea tropical, humana y contemporanea
- conserve capacidad de conversion
- ordene Eventos y Full Day como lineas secundarias

## Usuarios
- Pareja o familia buscando hospedaje
- Visitante que explora una escapada corta
- Cliente que evalua Wakaya como venue para evento

## Journey
Inspirarse -> entender la propuesta -> revisar habitaciones ->
abrir detalle -> iniciar prereserva.

## Direccion elegida
- Variante base: `C`
- Nombre de direccion: `Tropical Modern Warmth`
- Hero: personas viviendo la experiencia
- Booking: barra flotante integrada
- Tipografia: serif editorial + sans limpia

## Principios de experiencia
- La fotografia debe sostener emocion real, no decoracion vacia.
- La jerarquia debe favorecer hospedaje antes que servicios secundarios.
- El sitio debe sentirse premium, pero no frio ni corporativo.
- La conversion debe estar visible sin romper la atmosfera de marca.

## Hipotesis
Una home y paginas internas con tono premium tropical, booking claro y
habitaciones curadas ayudaran a que Wakaya se perciba mas confiable y mas
valiosa sin perder cercania.

## Metricas de validacion cualitativa
- la home se siente propia de Wakaya y no de un template generico
- el camino hacia prereserva se entiende sin esfuerzo
- la relacion hospedaje / eventos / full day se percibe ordenada
- el sitio transmite valor suficiente para sostener una tarifa premium

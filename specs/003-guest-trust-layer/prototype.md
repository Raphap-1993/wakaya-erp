# Prototipo - Wakaya Guest Trust Layer

[README principal](../../README.md) | [Specs](../README.md)

## Objetivo del prototipo
Validar la claridad de la capa de confianza sin rediseñar el sistema visual
base ya aprobado para la web pública de Wakaya.

## Patron visual elegido
Extensión editorial del sitio público existente.

## Sistema visual e identidad
- Se conserva la identidad visual pública de Wakaya.
- La variación principal está en contenido, jerarquía y confianza visible.

## Anatomia
- Footer con shortcuts de confianza.
- Página de políticas con secciones ancla y texto escaneable.
- Página pet friendly con condiciones claras y tono amigable.
- Formulario público de reclamos con confirmación y constancia.

## Estados UI que el prototipo debe cubrir
- success
- validation-error
- submission-success
- empty admin
- error recuperable

## Responsive
- footer accesible en mobile
- booking band usable en mobile
